# Agent 3 — ML Predictor (CIRO)

## Purpose
Agent 3 manages the machine learning prediction models within the CIRO system. It generates 30-day forward-looking predictive risk models covering urban flooding and heatwave risks for 8 monitored Pakistani cities across 5 provinces.

---

## Machine Learning Models

### 1. XGBoost Classification Engine
The flood classification engine evaluates live feature vectors to predict binary flood risk (Low/Medium/High thresholds).
- **Dataset Scale**: Trained on **1,572 historical samples** containing **60 major flood events** in Pakistan's urban history.
- **Features Ingested**:
  - `temp_avg` / `temp_max`: Running temperature metrics.
  - `precip_monthly_cumulative`: 30-day cumulative rainfall.
  - `month`: Captures seasonal monsoon patterns.
  - `province_encoded`: Encodes geographical and administrative layers.
  - `antecedent_moisture`: Soil water saturation indicators.

### 2. Prophet Forecasting Engine
Prophet forecasts temperature and rainfall variables into the future, particularly for days 17 through 30.
- **Model Distribution**: Maintains **12 custom Prophet models** (6 provinces × temperature + rainfall models).
- **Training Baseline**: Trained using **22 years of daily meteorological data** extracted directly from Google Earth Engine (GEE).

---

## GloFAS Discharge Integration
For maximum prediction accuracy, the ML Predictor integrates daily river discharge projections directly from the Global Flood Awareness System (GloFAS). The daily discharge data modulates the statistical probability generated by the XGBoost classifier, raising risk flags if upstream discharge rates exceed critical thresholds.

---

## Prediction Confidence Tiers
To ensure operational readiness while managing forecast uncertainty, predictions are categorized into confidence tiers based on the time horizon:

| Forecast Horizon | Confidence Tier | Data Integrity | Description |
| :--- | :--- | :--- | :--- |
| **Days 1–7** | **HIGH** | Open-Meteo + GloFAS + Live Telemetry | Extremely precise; immediate tactical triggers. |
| **Days 8–16** | **MODERATE** | Open-Meteo 16-Day Weather Forecasts | High accuracy; useful for operational scheduling. |
| **Days 17–30** | **LOW** | Prophet Seasonal Forecasting Engine | Long-term warning indicators for early staging. |

---

## PMD Heat Engine
Heatwave forecasts are monitored by a custom PMD Heat Engine. It defines dynamic temperature thresholds for each zone based on historical baselines (e.g., Karachi heatwave threshold triggers at lower absolute temperatures than Multan due to coastal humidity and historical baselines).

## API Endpoints (`/api/v1/agent3/*`)

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/status` | Retrieves model load states (XGBoost/Prophet status). |
| `POST` | `/predict/{zone_id}` | Generates 30-day forward-looking predictive risk models. |
| `GET` | `/model/info` | Returns model metadata, including accuracy scores and training samples. |
| `POST` | `/backtest` | Executes a retrospective test of predictions against historical database data. |
